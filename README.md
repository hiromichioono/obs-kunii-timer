# obs-kunii-timer

OBS Studio 用のストップウォッチ & YouTube チャットオーバーレイです。

---

## ファイル構成

```
obs-kunii-timer/
├── timer.lua          # OBS Lua スクリプト（タイマー）
├── chat_server.py     # YouTube チャット取得・WebSocket 配信サーバー
├── chat_overlay.html  # OBS ブラウザソース用チャットオーバーレイ
└── logs/              # チャットログ保存先（.gitignore 対象）
```

---

## タイマー（timer.lua）

OBS のテキストソースに経過時間を `MM:SS` 形式でリアルタイム表示します。

### 特徴

- `os.time()` 差分方式を採用。OBS タイマーの発火遅延に依存しないため、長時間運用でも誤差が蓄積しません。
- キックオフ / 一時停止 / リセット / 時刻同期に対応。
- Linux・Mac のテキストソース対応（`text_ft2_source`）。

### インストール

1. OBS Studio を起動する。
2. メニューから「ツール」→「スクリプト」を開く。
3. `+` ボタンで `timer.lua` を追加する。

### 使い方

1. スクリプトを追加すると設定パネルが表示されます。
2. 「表示するテキストソース」で時間を表示したいテキストソースを選択します。
3. 「ステータスのテキストソース」でステータスを表示したいテキストソースを選択します。
4. ボタンで操作します。

| ボタン | 動作 |
|--------|------|
| キックオフ / 再開 | タイマーを開始・再開する |
| 一時停止 | タイマーを一時停止する |
| リセット (00:00) | タイマーを停止して `00:00` に戻す |
| 時刻を同期 | 指定した分・秒にタイマーをセットする |

### ステータス表示

| 状態 | テキストソースの表示 |
|------|---------------------|
| 動作中 | `LIVE` |
| 一時停止中 | `BREAK`（1 秒ごとに点滅） |

### テキストソースの作成時の注意

- テキストソースのプロパティで **「ファイルから読み込む」のチェックを外す**こと。チェックが入っているとスクリプトからの更新が無効になり、タイマーが表示されません。
- シーンからソースを削除して再追加する場合は「**既存のソースを追加**」を使う。新規作成すると「ファイルから読み込む」がデフォルトで有効になる場合があります。
- スクリプトパネルで **ソース名を選択してから** 操作すること。選択前はタイマー・ステータスともに動作しません。

### 注意

- OBS を再起動すると経過時間はリセットされます。
- タイマーの精度は `os.time()` の 1 秒単位です。

## チャットオーバーレイ（chat_server.py + chat_overlay.html）

YouTube Live のチャットをリアルタイムで OBS に表示するシステムです。

### 構成

```
YouTube Live
    ↓ pytchat
chat_server.py ──→ WebSocket (ws://localhost:8765) ──→ chat_overlay.html（OBSブラウザソース）
    ↓
logs/YYYY/YYYY_MM/chat_log_YYYYMMDD_HHMM.txt
```

### 動作環境

- Python 3.8 以上
- `pip install pytchat websockets`

### 使い方

**1. サーバーを起動する**

```bash
python chat_server.py "https://www.youtube.com/watch?v=xxxxxx"
```

URL を省略すると入力プロンプトが表示されます。

**2. OBS にブラウザソースを追加する**

1. OBS でソース追加 → 「ブラウザ」
2. 「ローカルファイル」にチェックを入れ、`chat_overlay.html` を選択
3. 幅・高さをキャンバスと同じ解像度に設定（例：1920×1080）

**3. 確認**

- オーバーレイ右下が **LIVE（緑）** になれば接続成功
- チャットが右下に最新 6 件スライドイン表示されます

### 注意

- `chat_server.py` と OBS を同じマシンで起動してください。
- ログは `logs/` フォルダに自動保存されます（`.gitignore` 対象）。
